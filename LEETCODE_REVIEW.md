# Nudge for LeetCode Problem Review

## Overview

Nudge is a spaced repetition study reminder that helps you maintain and improve your LeetCode problem-solving skills through scientifically-proven review intervals. Instead of forgetting problems you've already solved, Nudge reminds you exactly when to review them for optimal retention and mastery.

## The Problem with LeetCode Practice

When you solve a LeetCode problem:
- You feel confident immediately after solving it
- Days later, you've already forgotten the approach
- Weeks later, solving a similar problem feels like starting from scratch

**Result:** You practice more problems, but retain less knowledge.

## The Solution: Spaced Repetition

Nudge uses the **forgetting curve** principle - you review problems at scientifically-optimized intervals:

- **Day 1**: Solve the problem initially
- **Day 3**: Review the approach (2 days later)
- **Day 7**: Review the solution again (4 days later)
- **Day 14**: Another review (7 days later)
- **Day 30**: Deep review (16 days later)
- **Day 60**: Challenge yourself again (30 days later)
- **Day 120**: Final reinforcement (60 days later)

After 120 days, the problem is marked as **"mastered"** and continues at 120-day intervals for long-term retention.

## How to Use Nudge for LeetCode

### 1. Add a Problem After Solving It

When you successfully solve a LeetCode problem:

1. Open Nudge
2. Click **"Add Item"**
3. Enter the problem details:
   - **Name:** e.g., "LeetCode #200 - Number of Islands"
   - **Tags:** Use tags like:
     - `Medium` (difficulty)
     - `BFS` or `DFS` (algorithm type)
     - `Graph` (topic)
     - `Week-1` (study group)

Example:
```
Item Name: LeetCode #1 - Two Sum
Tags: Easy, Hash Map, Array
```

### 2. Daily Review Routine

Every morning, open Nudge and check what's due:

- **Red highlighted items:** Problems overdue for review
- **Yellow highlighted items:** Problems due today
- Review each problem and solve it again (or trace through the approach)
- Click **"Mark as Reviewed"** to schedule the next review

### 3. Review Tips

When reviewing a problem:

- **First review (Day 3):** Re-read the problem, think about your approach, code it from scratch
- **Second review (Day 7):** Solve it again without looking at notes
- **Third review (Day 14):** Solve with slight variations or different approaches
- **Later reviews:** Challenge yourself with optimizations or different implementations
